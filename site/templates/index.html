Features

  • Tree browser

    Navigate your store directory tree with vim keys (hjkl) or arrows.

  • Masked & reveal

    Passwords are masked by default; reveal only on an explicit action.

  • Live OTP / TOTP

    Generate and copy one-time codes from otpauth:// URIs.

  • Copy & auto-clear

    Clipboard copies clear after 45s — only if the value is still yours.

  • Fuzzy search

    Live search across all entry paths and metadata.

  • Create / edit / delete

    Form-based with templates and notes — or raw $EDITOR edit.

  • CSPRNG generator

    Secure password generation with configurable length and charset.

  • Obsidiana & Oro

    A clean, dark, gold-accented theme across both apps.

How it works

Ichtaca does not replace pass. It wraps the pass CLI and gpg/gpg-agent, giving you a richer interface over the same ~/.password-store — no migration, no new data format.

Requirements

  • pass installed and on $PATH
  • gpg / gpg-agent with a configured key
  • An initialized store: pass init <gpg-key-id>
  • TUI only: a Nerd Font in your terminal
  • Platforms: macOS & Linux

Security

Designed to keep secrets off disk and out of logs.

  • Decryption is delegated to gpg/gpg-agent via pass — no decrypted data is written to disk.
  • In-memory secrets are zeroized on drop and redacted from debug output.
  • Passwords masked by default; clipboard auto-clears after 45 seconds with an ownership check.
  • Desktop app keeps plaintext in the Rust backend; only explicit reveal commands cross IPC. Strict CSP — no remote scripts or URLs.
  • No telemetry, no network connections (other than git if your store uses it).
